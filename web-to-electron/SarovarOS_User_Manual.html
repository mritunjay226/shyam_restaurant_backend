Getting Started

Everything you need to know before you begin using SarovarOS.

What is SarovarOS?

SarovarOS is a fully integrated Property Management System (PMS) built specifically for Sarovar Palace, Prayagraj. It combines hotel front-desk operations, restaurant and café point-of-sale, grocery store management, banquet and events booking, HR and payroll, and business analytics into a single web-based platform available as a desktop application.

💡
Cloud + Desktop: SarovarOS runs in the cloud at shyam-restaurant-backend.vercel.app and is also available as the SarovarOS Desktop App (Windows Installer). Both are identical in features.

🔐 Login & User Roles

SarovarOS uses a secure PIN-based login. When you open the app you will see a keypad. Enter your PIN to access your role-specific dashboard.

Role PIN Access Level Typical User
ADMIN / Boss 1111 Full access to all modules Owner / General Manager
Manager Set by Admin Operations, reports, staff Hotel Manager
Kitchen Set by Admin Kitchen KDS only Head Chef / Cook
⚠️
Keep PINs confidential. The Boss PIN provides unrestricted access to financial data, salary records, and system settings. Change default PINs immediately after first setup via Settings → Staff.

How to Change Room Status

Navigate to Rooms

Click Rooms in the left sidebar. You will see a grid/list of all rooms.

Select a Room

Click on the specific room card you want to update.

Update Status

Use the status dropdown or action buttons to change the room status (e.g., mark as Clean, Dirty, or Maintenance).

Save

Changes are saved automatically and reflected on the Dashboard in real-time.

Module 03

📅
Bookings & Reservations

Create and manage room reservations for walk-in guests and advance bookings. Handle check-ins, check-outs, and booking modifications.

➕

New Booking

Create a new reservation by selecting room, guest, check-in/out dates, and rate. System checks availability automatically.

🚪

Walk-in Check-in

For immediate arrivals without prior reservation — fill guest details and assign an available room instantly.

✏️

Modify Booking

Extend stay, change room, update guest information, or add extra charges to an existing booking.

❌

Cancel Booking

Cancel a future reservation. The system will automatically return the room to Available status.

Booking Workflow

Go to Bookings page

Click Bookings in the sidebar. You'll see all current and upcoming reservations.

Click "New Booking"

A booking form/panel will open on the right side of the screen.

Fill Guest Details

Enter guest name, phone number, and identity proof (Aadhaar, PAN, Passport, etc.).

Select Room & Dates

Choose the required room type, available room number, check-in date, and expected check-out date.

Confirm & Check-in

Review the rate (₹/night), total amount, and confirm. For check-in, the room status changes to Occupied immediately.

ℹ️
Advance Booking: You can create bookings for future dates. They appear in the booking list with status "Upcoming" until the check-in day arrives.

Module 05

🧾
Billing & Checkout

Generate the final consolidated bill for a guest and process their checkout. The billing module consolidates room charges, room service, restaurant tabs, and any other add-ons.

🏨

Room Billing

Shows all currently checked-in rooms. Click a room card to open the folio and generate the final bill.

🍴

Table Service Billing

Switch to "Table Service" tab to close out a restaurant table's running tab with full itemised details.

🖨️

Print / Digital Bill

Generate a professional printable invoice or share a digital receipt with the guest via WhatsApp.

💳

Payment Modes

Accept payment via Cash, UPI, Card (Razorpay), or mark as Credit. All modes are tracked separately.

Checkout Process

Go to Billing

Click Billing in the sidebar under the Finance section.

Select the Room / Table

You will see occupied rooms as cards. Click the room the guest is checking out from (e.g., #102 — Standard).

Review the Folio

The system shows all charges: room nights, any restaurant orders, extra services. Add or remove line items if needed.

Select Payment Method & Settle

Choose payment method (Cash / UPI / Card / Credit) and click Settle Bill. Room status automatically updates to Dirty.

Print or Share Receipt

Print the GST invoice or send a digital copy to the guest. Transaction is recorded in Reports.

⚠️
GST Notice: All bills are GST-compliant. Ensure room rates are correctly configured in Settings so that the correct GST rate is applied automatically to the invoice.

Module 06

🍽️
Restaurant POS

A full-featured Point of Sale for the hotel's main restaurant. Manage dine-in tables, takeaway orders, and delivery from a single interface.

Order Types Supported
TypeDescription
Table 1–8Dine-in orders for a specific table. Orders go live to Kitchen KDS.
Walk-inCounter orders for guests who walk in without a table reservation.
TakeawayPacked orders for guests taking food to go.
DeliveryOrders placed for delivery to rooms or outside addresses.
Room ServiceIn-room dining orders linked to a specific room booking.

Placing a Restaurant Order

Select Table or Order Type

At the top of the Restaurant POS, tap the table number tab (e.g., Table 3) or select Walk-in / Takeaway / Delivery.

Browse the Menu

Use the category tabs (Breakfast, Starters, Main Course, Beverages, etc.) to browse available dishes. Search by name using the search bar.

Add Items

Click the + button on any dish card to add it to the current order. Quantity adjusts with +/- controls.

Add Special Notes (Optional)

Add cooking instructions or guest preferences as a note on individual items.

Place Order (KOT)

Press Place Order / Print KOT. The kitchen instantly sees the ticket on the Kitchen Display System.

Settle Bill

Once the meal is done, press Bill to generate the GST invoice and process payment.

💡
Active KOTs Button: The "Active KOTs" button in the top-right corner of the Restaurant POS shows all currently running kitchen tickets so you can track which tables have pending orders.

Module 08

🔥
Kitchen Display System (KDS)

The Kitchen KDS is a digital replacement for printed kitchen tickets. It shows every active food order in real-time so kitchen staff can prepare and dispatch dishes efficiently.

⏱️

Live Order Feed

Orders appear on the KDS screen the moment they are placed from the Restaurant or Café POS — no paper tickets needed.

🍴☕

Restaurant / Café Toggle

Use the toggle buttons at the top-right to switch between Restaurant and Café order queues.

✅

Mark as Ready

Kitchen staff marks each order or individual item as ready, updating the waiter's view in real-time.

🔔

All Caught Up

When no orders are pending, the screen displays "All caught up! No pending orders." — a helpful status indicator.

⚠️
For Kitchen Staff Only: The KDS is designed to be displayed on a dedicated screen in the kitchen. Avoid using it for billing or POS operations. Only the Kitchen user role is needed to operate this screen.
💡
Mount a tablet or monitor near the cooking station and keep the KDS page open full-screen for best results. The system auto-refreshes in real-time — no manual refresh needed.

🗓️
Staff Attendance

Mark and track daily attendance for all staff members. Navigate by month to review historical attendance records.

How the Attendance Page Works

The Attendance page shows today's date and a list of all registered employees. For each employee, you mark their attendance status for the day.

Status ButtonMeaning
✓ PRESENTEmployee is at work today (full day)
ABSENTEmployee did not come to work
HALF DAYEmployee worked only half the day

Marking Daily Attendance

Open Attendance

Click Attendance in the sidebar. The page shows today's date and the daily manpower status header.

Review Employee List

All registered employees are listed with their role (Admin, Manager, Kitchen, etc.).

Click the Appropriate Status

For each employee, click Present, Absent, or Half Day.

Save Attendance

Press the purple SAVE ATTENDANCE button at the bottom. Attendance is recorded and feeds into the Payroll module automatically.

💡
Use the Month navigation arrows (< APR 2026 >) at the top of the page to view or correct attendance records from previous months.
ℹ️
Attendance data feeds directly into the Payroll module. Each employee's attendance summary (days present / total days) is displayed in the Payroll ledger and used to calculate net salary.

Calculate, review, and disburse monthly salaries for all staff. The payroll module automatically considers attendance records and any salary advances.

Processing Monthly Payroll

Navigate to Payroll

Click Payroll in the sidebar. The current month is selected by default.

Select Month

Use the MONTH dropdown to switch months if processing payroll for a previous period.

Review Each Employee

Check attendance days, baseline salary, and any pending advance amounts for each staff member.

Mark Salary as Paid

Once salary is disbursed (cash or bank transfer), click the employee row and mark status as PAID. This updates the Summary Cards.

⚠️
Important: Ensure all attendance is saved before processing payroll. The Net Final Pay calculation is based on attendance days. Missing attendance will result in incorrect salary calculations.

Troubleshooting

Common issues and how to resolve them quickly.

ProblemCauseSolution
Page not loading Poor internet connection or Vercel outage Check internet connection. Wait 60 seconds and refresh with F5.
Wrong PIN / Login fails Incorrect PIN entered Double check PIN. Default Boss PIN is 1111. Contact Admin to reset other PINs via Settings → Staff.
Bill shows ₹0 for food Menu item prices not configured Go to Settings → Menu and update the price for each item.
Room not showing as Occupied after check-in Booking not confirmed / syncing delay Refresh the Rooms page. Verify booking was saved in Bookings module.
KDS not showing new orders KDS page not in focus or auto-refresh paused Refresh the Kitchen KDS page with F5. Ensure stable internet.
Store counter shows "Terminal Not Ready" Counter not selected or session expired Click Store → select Counter again to re-initialise the terminal session.
Payroll shows 0 salary Salary baseline not set for employee Go to Settings → Staff → click the employee and set their monthly salary.
GST wrong on bill GST rate not configured for room/service type Go to Settings → GST Settings and confirm the correct slab is assigned.
💬
Need Further Help? Contact your system administrator or the development team. Always mention the specific page name and action you were performing when the issue occurred to speed up resolution.
